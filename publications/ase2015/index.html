From Requirements to Source Code: A Model-Driven Engineering Approach for RESTful Web Services

This page keeps the files required for reproducing our case study in paper:
From Requirements to Source Code: A Model-Driven Engineering Approach for RESTful Web Services
submitted to the Special Issue: Realizing AI Synergies in Software Engineering in the Journal of Automated Software Engineering

The case study involves project Restmarks, a service that allows users to store and retrieve online their bookmarks, share them with other users and search for bookmarks by using tags.
The elements of the project include:

  • Functional requirements, available here.
  • Storyboards, available here.
  • The YAML file, produced by the aggregated ontology upon instantiating it from multimodal types of requirements, available here.
  • The CIM, constructed using as input the YAML file, available here.
  • The PIM, constructed using as input the CIM, available here.
  • The PSM, constructed using as input the PIM, available here.
  • The produced code of the service, constructed using as input the PSM, available here.

Finally, the full definition of the PIM/PSM meta-models and the M2M transformations described in the manuscript can be found at deliverable D2.2 of S-CASE, available here.
